2. GitHub创建个人名字的仓库

这里只需要注意仓库名应该是 用户名.github.io ，且设置为public即可。

3. 安装hexo

在电脑里创建一个空文件夹Blog（别的名字也可以），hexo框架与以后发布的网页都会存在这里面。

创建好，进入文件夹，在当前位置打开cmd，使用如下命令安装hexo：

常用的hexo命令：

  • hexo s等价于 hexo server ：Hexo 会监视文件变动并自动更新，除修改站点配置文件外,无须重启服务器,直接刷新网页即可生效。

  • hexo g 等价于 hexo generate ：生成静态网页 (执行 $ hexo g 后会在站点根目录下生成public>文件夹, hexo会将"<font /blog/source/" 下面的.md后缀的文件编译为.html后缀的文件,存放在"/blog/public/ " 路径下)

  • hexo d 等价于 hexo deploy ：将本地数据部署到远端服务器(如github)

  • hexo clean ：

  • 清除缓存 ,网页正常情况下可以忽略此条命令,执行该指令后,会删掉站点根目录下的public文件夹

    hexo server #Hexo会监视文件变动并自动更新，无须重启服务器 hexo server -s #静态模式 hexo server -p 5000 #更改端口 hexo server -i 192.168.1.1 #自定义 IP

5. 绑定域名

登录阿里云，进入管理控制台的域名列表，找到自己的个性化域名，进入解析。

添加解析：需要添加两个记录

  • 第一个主机记录是@，第二个主机记录是www，记录值都是博客地址（"name".github.io）或博客IP地址（cmd ping博客地址可得到）

登录GitHub，进入github名的仓库，在settings的pages里的custom domain输入自己的域名。

进入本地博客文件夹，在blog/source目录中，创建一个记事本文件，输入自己的域名，建议不带www，若有，则以后每次访问都需要加www，没有的话可加可不加都能访问。保存为CNAME，要保存成所有文件而不是.txt文件。
